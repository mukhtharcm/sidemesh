Sidemesh is a small daemon and a smaller app. Run the daemon on any machine where Codex already runs — your MacBook, a lab box, a rented VPS, that Mac mini you forgot in a closet — and the app reaches each of them over your own network. No pairing server. No cloud broker. No account to create.

(02) how it wires together

A tiny daemon per machine. One app for all of them.

Each machine runs a small Node daemon that owns one codex app-server process over stdio and exposes an HTTP + WebSocket API behind a bearer token. Run as many daemons as you like. The app talks to each one directly — nothing in the middle.

  ┌──────────────┐                          ┌──────────────┐                  ┌──────────────────┐
  │     app      │  ── bearer token ──▶   │    daemon    │  ── stdio ──▶    │  codex           │
  │  phone / mac │      HTTP + WS           │    (node)    │                  │  app-server      │
  │              │  ◀──────────────────   │              │  ◀──────────       │  (your files)    │
  └──────────────┘                          └──────────────┘                  └──────────────────┘

Pairs nicely with Tailscale, a Cloudflare Tunnel, plain ssh -R, or whatever you already use to reach your boxes.

(03) things sidemesh does not try to do

A short honest list.

Being small means being opinionated. Here's what lives outside the scope:

  • install or manage Codex for you — bring your own
  • sync files across machines — your workspace lives on the daemon's disk, and that's the feature
  • store anything in the cloud — there is no cloud
  • replace your editor — go use Zed, or vim, or whatever
  • add a pairing server or a third-party relay — you already own a network
