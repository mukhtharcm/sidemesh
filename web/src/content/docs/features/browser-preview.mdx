Browser preview captures screenshots of a headless browser running on the host and streams them to the client as a live image feed.

## How it works

The daemon launches a headless browser (Chromium via Playwright or similar), navigates to a URL, and periodically captures screenshots. Images are pushed to the client over the live event WebSocket.

## Use cases

- See what the agent sees when it uses a browser tool
- Review visual changes without port-forwarding a dev server
- Verify rendering on the host's browser engine
- Catch UI bugs the agent might miss

## Limitations

- Screenshots are lower fidelity than a real browser
- Interaction is limited; for full control, use port forwarding
- The browser runs on the host, not the client, so it sees the host's network

## Comparison with port forwarding

Browser preview is for quick visual checks. Port forwarding is for full interaction. See [Browser vs Port Forwarding](/comparisons/browser-vs-port-forwarding/) for a detailed comparison.
